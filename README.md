# 🏥 Consultorio AdminPlan - Sistema de Gestión de Turnos (Frontend)

Este repositorio contiene la interfaz administrativa (Frontend) para el sistema de gestión de turnos médicos. El proyecto es una Single Page Application (SPA) conceptual que consume una **API REST** desarrollada en Java con Spring Boot.

## 🚀 Live Demo
Puedes ver la aplicación funcionando en vivo aquí:
👉 **[Consultorio AdminPlan en Vivo](https://hughob36.github.io/Consultorio_AdminPlan/index.html)**

---

## ✨ Características Principales
- **Dashboard Administrativo:** Interfaz profesional para la supervisión de operaciones.
- **Gestión de Turnos:** Visualización y asignación de citas médicas consumiendo datos dinámicos.
- **Control de Acceso:** Lógica de visualización condicionada por la presencia de **JWT**.
- **Diseño Responsive:** Adaptabilidad total a móviles y tablets.
- **Integración con Backend:** Diseñado específicamente para trabajar en conjunto con la [API de Gestión de Turnos](https://github.com/hughob36/Consultorio_Turnos).

## 🛠️ Tecnologías Utilizadas
- **HTML5 & CSS3:** Estructura y estilos modernos.
- **JavaScript (ES6+):** Manipulación del DOM y consumo de servicios con `Fetch API`.
- **Bootstrap 5:** Framework de diseño para una interfaz ágil y responsiva.
- **Font Awesome & Bootstrap Icons:** Iconografía técnica y de usuario.
- **GitHub Pages:** Hosting del despliegue frontend.

## 🔌 Cómo probar la integración
Para ver el funcionamiento completo de las tablas y registros:
1. Debes tener el **Backend** corriendo localmente o en un servidor accesible.
2. La interfaz detectará el token de autenticación en el `localStorage` para habilitar las opciones de administración avanzada.
